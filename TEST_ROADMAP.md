# Test Roadmap — OpenCart Admin Panel Automated UI Testing

## Goal

Build and maintain a stable, readable, and scalable automated UI test suite
for critical OpenCart Admin Panel workflows.

---

## Phase 1 — Core automation (Current)
- Automate key admin workflows
- Establish Page Object Model structure
- Ensure tests are readable and repeatable
- Validate product and order management flows

---

## Phase 2 — Coverage expansion
- Add negative admin scenarios
- Validate mandatory field handling
- Extend coverage for bulk actions and edge cases

---

## Phase 3 — Stability improvements
- Reduce flaky admin UI tests
- Improve waits and synchronisation
- Refactor shared admin actions and utilities

---

## Phase 4 — Execution improvements
- Enable reliable cross-browser execution
- Improve reporting and screenshots
- Prepare for CI-ready execution setup

---

## Maintenance
- Update tests when admin UI changes
- Keep documentation in sync with automation coverage
